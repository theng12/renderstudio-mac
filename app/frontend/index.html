Release notes

What's New

Checking installed version...

0.7.3 / Supervised renders and clearer controls

  • Automatic updates now offer only genuinely newer releases, never a downgrade when this Mac is ahead.
  • FFmpeg sends durable heartbeats and is stopped, force-killed when necessary, and fully reaped after a timeout or cancellation before partial output cleanup continues.
  • The Pinokio sidebar now shows Updating and Resetting states with What's New always available, while dashboard labels and controls meet the shared readable minimum size.

0.7.2 / Actionable render failures

  • Failed FFmpeg and FFprobe steps now include a bounded diagnostic tail instead of only an opaque step number.
  • Worker-local data paths are scrubbed before diagnostics travel through Studio Hub and Story Studio.
  • Release checks now require the current version to appear in both the changelog and this What's New view.

0.7.1 / Automatic local-backup protection

  • Verified render copies now default to three days with an 80 GB hard cap and hourly enforcement.
  • Active, pinned, and not-yet-returned renders remain protected; eligible verified copies are removed oldest first.
  • Modern Save policy, Clean now, retention, capacity, and free-reserve controls are available locally and through Studio Hub.

0.7.0 / Safe automatic updates

  • Choose Off, Notify only, or Automatic directly in Render Studio, with a visible daily or weekly maintenance schedule.
  • Running and queued renders defer updates without interruption; Update after current work survives page closes and worker restarts.
  • Updates enforce the fixed repository, clean fast-forward, disk, dependency, import, health, exact-version, restart, and rollback checks used by sibling Studios.

0.6.0 / Complete Video Assembly on workers

  • Render workers now receive every enabled Video Assembly asset before rendering: title and outro cards, timeline clips, music, ambience, overlays, frame media, subtitle files, and compilation backgrounds.
  • Workers now apply text cards, chapter titles, chroma key, timed overlay layers, animated subtitles, compilation layouts, and the same finishing controls before returning a verified final video.
  • MP4, MOV, and WebM exports now keep their selected container all the way through the worker render and verified download.

0.5.0 / Full visual assembly on workers

  • Render workers now receive and verify title images, logos, presentation backgrounds, and the same finishing effects used by Story Studio.
  • Color grading, vignette, film grain, cinematic letterbox bars, and branded overlays render locally on the selected worker before the verified copy returns.
  • Worker discovery now advertises the richer Video Assembly capability set so Story Studio can route eligible finished videos without silently dropping visual settings.

0.4.0 / Releases you can trust

  • The installed version is always visible and comes from the same release file used by the worker and Studio Hub.
  • What's New now follows the version actually installed on this Mac, so every new release is surfaced once.
  • A quiet update notice appears when a newer release is published; use Update in the Pinokio sidebar to install it.

0.3.3 / Reliable dashboard

  • The dashboard keeps working after old renders are cleaned up, and lifetime video totals survive retention.
  • Cancelled renders no longer leave leftover files counting against disk usage.

0.3.2 / Accurate version

  • The worker now reports its real version to Studio Hub and this dashboard instead of a stale build number.

0.3.1 / Live credentials

  • Fleet credential saves and rotations take effect immediately, with no worker restart needed.

0.3.0 / Worker intelligence

  • Clear worker-online and authenticated Studio Hub connection status with a manual connection test.
  • Live episode progress plus durable recent and lifetime render reporting.
  • Hardware, encoder usage, storage, retention, and Hub settings in one dashboard.

0.2.0 / Always available

  • Optional startup service, crash recovery, health watchdog, dedicated app icon, and service-aware updates.

0.1.0 / First render

  • Verified local assets, VideoToolbox encoding with CPU fallback, complete output validation, and safe acknowledged retention.
